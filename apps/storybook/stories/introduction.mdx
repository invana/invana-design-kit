# Invana Design System

Welcome to the Invana Design System - a comprehensive, theme-aware component library built with React, TailwindCSS v4, and TypeScript.

## ğŸ¨ Design Philosophy

Our design system provides:

- **Multiple Themes**: Pre-built themes (Invana, Tailwind, Vite) with light and dark variants
- **Type-Safe Components**: Fully typed React components with TypeScript
- **Tailwind v4**: Leveraging the latest TailwindCSS features with `@theme` and CSS variables
- **Accessibility**: ARIA-compliant components built on Radix UI primitives
- **Customizable**: Easy to extend and customize for your brand

### Basic Setup

1. **Import CSS in your app's entry point:**

```css
/* app.css or globals.css */
@import "tailwindcss";

/* Define your source paths */
@source "./src/**/*.{js,ts,jsx,tsx}";

/* Import the design system */
@import "@invana/styling/index.css";

/* Import theme variants */
@import "@invana/styling/themes/default.css";
@import "@invana/styling/themes/tailwind.css";
@import "@invana/styling/themes/vite.css";
```

2. **Use components in your React app:**

```tsx
import { Button, Card, Badge } from '@invana/ui';

function App() {
  return (
    <Card>
      <h1>Welcome</h1>
      <Button variant="primary">Get Started</Button>
    </Card>
  );
}
```

## ğŸ­ Theme System

### Available Themes

Our design system includes three carefully crafted themes:

1. **Invana (Default)** - Green accents, clean and modern
2. **Tailwind** - Blue/cyan accents, inspired by tailwindcss.com
3. **Vite** - Purple/violet accents, inspired by vite.dev

Each theme includes **light** and **dark** variants, totaling 6 theme options.

### Using Themes

#### Option 1: JavaScript/TypeScript API

```tsx
import { applyTheme, themes } from '@invana/styling/themes';

// Apply a specific theme variant
applyTheme('tailwind-dark');

// Available theme IDs:
// - 'default-light', 'default-dark'
// - 'tailwind-light', 'tailwind-dark'
// - 'vite-light', 'vite-dark'

// Get all available themes
console.log(themes);
```

#### Option 2: HTML Data Attribute

```html
<!-- Set theme on root element -->
<html data-theme="vite-light">
  <body>
    <!-- Your app content -->
  </body>
</html>
```

#### Option 3: Dynamic Theme Switching

```tsx
import { useState } from 'react';
import { applyTheme } from '@invana/styling/themes';

function ThemeSwitcher() {
  const [theme, setTheme] = useState('default-light');

  const handleThemeChange = (newTheme: string) => {
    setTheme(newTheme);
    applyTheme(newTheme);
  };

  return (
    <select value={theme} onChange={(e) => handleThemeChange(e.target.value)}>
      <option value="default-light">Invana Light</option>
      <option value="default-dark">Invana Dark</option>
      <option value="tailwind-light">Tailwind Light</option>
      <option value="tailwind-dark">Tailwind Dark</option>
      <option value="vite-light">Vite Light</option>
      <option value="vite-dark">Vite Dark</option>
    </select>
  );
}
```

## ğŸ¨ Creating Custom Themes

### Step 1: Create Theme CSS File

Create a new CSS file for your theme (e.g., `my-theme.css`):

```css
/**
 * My Custom Theme
 * Description: Your custom brand colors
 */

/* Light variant */
[data-theme="my-theme-light"],
.theme-my-theme-light {
  /* Raw HSL values for opacity modifiers */
  --background: 0 0% 100%;
  --foreground: 220 10% 10%;
  --primary: 200 90% 50%;
  --primary-foreground: 0 0% 100%;
  --secondary: 200 20% 95%;
  --secondary-foreground: 200 80% 30%;
  --muted: 200 10% 95%;
  --muted-foreground: 200 10% 45%;
  --accent: 280 80% 55%;
  --accent-foreground: 0 0% 100%;
  --destructive: 0 80% 55%;
  --destructive-foreground: 0 0% 100%;
  --border: 200 15% 85%;
  --input: 200 15% 90%;
  --ring: 200 90% 50%;
  --card: 0 0% 100%;
  --card-foreground: 220 10% 10%;
  --popover: 0 0% 100%;
  --popover-foreground: 220 10% 10%;

  /* Full HSL colors for direct use (optional) */
  --color-background: hsl(0 0% 100%);
  --color-foreground: hsl(220 10% 10%);
  --color-primary: hsl(200 90% 50%);
  /* ... add all color variants */
}

/* Dark variant */
[data-theme="my-theme-dark"],
.theme-my-theme-dark {
  --background: 220 15% 8%;
  --foreground: 0 0% 95%;
  --primary: 200 90% 60%;
  --primary-foreground: 220 15% 10%;
  /* ... define all dark colors */
}
```

### Step 2: Register Your Theme

Update the theme configuration in `packages/styling/src/themes.config.ts`:

```typescript
export const themes: Theme[] = [
  // ... existing themes
  {
    id: 'my-theme',
    name: 'My Custom Theme',
    description: 'Custom brand colors for my app',
    variants: [
      { id: 'my-theme-light', name: 'My Theme Light', mode: 'light', icon: 'sun' },
      { id: 'my-theme-dark', name: 'My Theme Dark', mode: 'dark', icon: 'moon' }
    ]
  }
];
```

### Step 3: Import Theme CSS

Add your theme to the main import list:

```css
/* In your app's CSS */
@import "@invana/styling/themes/my-theme.css";
```

## ğŸ› ï¸ Extending Themes

### Override Specific Colors

You can override specific theme colors without creating a full theme:

```css
/* Override primary color for all themes */
:root {
  --primary: 180 70% 45%;
  --primary-foreground: 0 0% 100%;
}

/* Override only for a specific theme */
[data-theme="default-light"] {
  --primary: 150 80% 40%;
}
```

### Add Custom Theme Variables

Extend themes with your own variables:

```css
@theme {
  /* Add custom variables */
  --sidebar-width: 16rem;
  --header-height: 4rem;
  --custom-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}
```

### Per-Component Theming

Override colors for specific components:

```tsx
// Using inline styles with CSS variables
<Button 
  style={{ 
    '--primary': '280 80% 50%',
    backgroundColor: 'hsl(var(--primary))' 
  }}
>
  Custom Color Button
</Button>

// Or use Tailwind classes with arbitrary values
<Button className="bg-[hsl(280_80%_50%)]">
  Custom Color Button
</Button>
```

## ğŸ¯ Key Concepts

### CSS Variables vs Tailwind Classes

The design system uses **CSS variables** for theming and **Tailwind utility classes** for styling:

```tsx
// âœ… Good: Uses theme-aware CSS variables
<div className="bg-primary text-primary-foreground">
  Primary colored box
</div>

// âœ… Good: Uses Tailwind utilities
<div className="rounded-lg shadow-md p-4">
  Card content
</div>

// âš ï¸ Avoid: Hard-coded colors break theming
<div className="bg-blue-500 text-white">
  This won't change with themes
</div>
```

### Color Tokens

Our theme system uses semantic color tokens:

- `background` / `foreground` - Base page colors
- `primary` / `primary-foreground` - Primary actions
- `secondary` / `secondary-foreground` - Secondary actions
- `accent` / `accent-foreground` - Highlights and emphasis
- `muted` / `muted-foreground` - Subdued backgrounds and text
- `destructive` / `destructive-foreground` - Errors and warnings
- `border` / `input` / `ring` - UI element colors
- `card` / `card-foreground` - Card components
- `popover` / `popover-foreground` - Overlay components
